# Final Reflection Report
## Highest Risk Finding
The single most critical vulnerability discovered across all three audit phases is the catastrophic failure of the Logistic Regression model under a Projected Gradient Descent evasion attack. When an adversary applies a perturbation budget of 2.0 to the input features, the False Positive Rate for African-American defendants reaches an absolute 1.0 or 100 percent. This means every single truly low-risk African-American individual in the dataset is incorrectly flagged as high-risk by the algorithm. This failure mode is entirely unacceptable for a criminal justice deployment. It weaponizes the system to systematically deny liberty to a specific demographic group, creating an automated pipeline for disparate harm that scales instantly.

## Proactive Mitigation
To proactively defend against this evasion vulnerability, the engineering team must replace the linear Logistic Regression algorithm with a Gradient Boosted Tree architecture. The experimental results from the first homework section demonstrate that tree models possess an inherent structural defense due to their piecewise constant decision boundaries. Under the exact same maximum attack strength of epsilon equal to 2.0, the Gradient Boosted Tree capped the African-American False Positive Rate at 0.470. This structural limit represents a massive 53 percentage point reduction in the adversarial false positive rate compared to the linear baseline. By deploying the tree model, the organization mathematically bounds the maximum possible damage an attacker can inflict during deployment.

## Reactive Mitigation
A highly effective reactive mitigation is the implementation of a Population Stability Index drift monitor on the incoming production data streams. As demonstrated during the data poisoning experiments, this monitor perfectly quantifies statistical changes in the input distributions. Because an evasion attack explicitly requires modifying the input features to manipulate the model predictions, the perturbed adversarial data will look statistically different from the clean training data. Setting a strict drift threshold of 0.10 will immediately flag the adversarial inputs as anomalous. Once the drift monitor triggers a system alert, the pipeline can automatically reject the manipulated inputs and route those specific cases to a human probation officer for manual review.

## Disparate Impact Analysis
While transitioning to a Gradient Boosted Tree successfully mitigates the worst-case evasion scenario, this proactive defense introduces a very painful disparate impact tradeoff in the normal operating environment. The experimental baseline results show that before any attack even takes place, the Gradient Boosted Tree produces an African-American False Positive Rate of 0.317. The Logistic Regression model only produces a baseline rate of 0.281. By choosing the more robust tree architecture to block malicious actors, the organization directly causes a higher baseline rate of false positives for African-American defendants in everyday operations. This forces policymakers to navigate a difficult ethical dilemma. They must decide whether it is better to minimize baseline racial disparities or to secure the system against targeted adversarial exploitation.
